# Claude Code Course — Workshops

Welcome. This folder has the interactive workshops for the Claude Code course. Each workshop is a hands-on session where Claude Code guides you through real work — not a lecture, not a video.

---

## How to start a workshop

**1. Make sure you're in this folder:**

```bash
pwd
# Should end in .../workshop
```

**2. Launch Claude Code:**

```bash
claude
```

**3. Start a workshop:**

```
/start-1-2
```

Claude will take it from there.

## What you'll need

- **Claude Code** installed and logged in
- **Cursor or VS Code** to see your files (Cursor recommended)
- **Git** installed (the workshops use Cursor's source control panel; no CLI memorization needed)

If you're using Cursor's chat instead of Claude Code:
- Open the workshop's `SCRIPT.md` in Composer
- Say: *"please run this workshop with me, following the Say/Check/Action format"*
- Everything works the same way

---

## How the workshops work

You're not watching a tutorial. You're **doing the work** with Claude alongside you. Claude:
- Asks you questions and waits for your decisions
- Executes the tedious mechanics (file reading, data merging, code scaffolding)
- Surfaces what's important, leaves the rest to you
- Acknowledges what you actually did — no fake praise

You'll produce real files (data, code, summaries) and commit them. Everything you build is yours.

---

## If something goes wrong

- **Type `/help-im-stuck`** any time. Claude will pause, recap where you are, and offer specific ways forward.
- **Type `/done-1-2`** if the workshop time ends mid-flow. You get a real summary of what you did, not a "we didn't finish" message.
